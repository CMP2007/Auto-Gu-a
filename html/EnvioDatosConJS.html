Envio de datos de un formulario HTML mediante JavaScript

Esta claro que los fommularios HTML son capases de enviar solicitudes HTTP al servidor, sin embago existen algunos puntos negativos de estas consultas, principalmente las paguinas web basicas se basan en cargar la paguina completa, ya que el navegador carga el URL al que se hace referencia, en un envio de formulario estandar al enviar los datos se vuelve a cargar la paguina, sin embargo esto conlleva ciertas desventajas como por ejemplo un posible retraso proporcionado por la red, el cual siempre se puede apreciar como un pequeño parpadeo al enviar los datos.

Por otra parte al realizar un envio de datos en base a JavaScript esto se puede modificar de forma que el envio se realise de forma asincrona en segundo plano, a la vez que unicamente se refresquen los elementos necesarios de la interfaz de usuario son refrescados aliminando de esa forma los incombenientes de una consulta tradicional.

Para este tipo de consultas asincronas se utilizan protocolos como JSON o XML a su vez que el lenguaje JavaScript para crear el codigo con el que sea realizara esta opción, en esta guía se puede indagar más sobre este tema.
